# Skill: Source-Backed Research

Use this skill when external documentation, standards, release notes, API behavior, or current facts affect the answer.

## Source Priority

1. Official docs
2. Specifications and standards
3. Release notes and changelogs
4. Source repositories
5. Vendor support pages
6. Secondary sources only as supporting context

## Output

- Findings
- Sources
- Versions or dates
- Applicability to this repository
- Confidence
- Open questions
- Recommendation

## Rules

- Browse or fetch current docs when facts may have changed.
- Avoid unsupported claims.
- Separate facts from inference.
- Keep quotes short and necessary.
